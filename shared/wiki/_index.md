## How to use this wiki

**Before Stage 01:** Read `winning-patterns.md` and `archetypes/` — surface patterns before ideating from scratch.

**Before Stage 02:** Read `stack-pitfalls.md` — know what has burned the team before finalizing the stack.

**After an event:** Drop sources into `shared/sources/{event-slug}/`, then run the compile step (see `CLAUDE.md` §7).

## Compile step summary

The compile step reads new sources, updates affected pages, and records every change in `schema/provenance.md`. Contradictions with existing content are flagged in `schema/contradictions.md` — never silently overwritten.
